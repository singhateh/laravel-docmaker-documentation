Introduction

API Doc Generator is a powerful Laravel package that automatically generates beautiful, interactive API documentation from your existing codebase. It provides a testing interface that allows developers to try out API endpoints directly from the documentation.

Easy Setup

Get started in minutes with automatic endpoint discovery and documentation generation.

Interactive Testing

Test your endpoints directly from the documentation with a built-in API client.

Customizable

Fully customizable templates and styles to match your brand identity.

Installation

1

Install via Composer

Install the package through Composer by running the following command in your terminal:

composer require yourusername/api-doc-generator
2

Publish Assets

Publish the package assets to your application:

php artisan vendor:publish --tag=api-doc-generator-assets
3

Publish Config

Publish the configuration file to customize the package:

php artisan vendor:publish --tag=api-doc-generator-config

Configuration

After installation, you can configure the package by modifying the config/api-doc-generator.php file:

return [
    /*
    |--------------------------------------------------------------------------
    | Route Prefix
    |--------------------------------------------------------------------------
    |
    | This prefix will be added to all documentation routes.
    |
    */
    'route_prefix' => 'api-docs',
    
    /*
    |--------------------------------------------------------------------------
    | Middleware
    |--------------------------------------------------------------------------
    |
    | Middleware to apply to the documentation routes.
    |
    */
    'middleware' => ['web'],
    
    /*
    |--------------------------------------------------------------------------
    | Documentation Title
    |--------------------------------------------------------------------------
    |
    | The title that will be displayed in the documentation.
    |
    */
    'title' => 'API Documentation',
    
    /*
    |--------------------------------------------------------------------------
    | Documentation Description
    |--------------------------------------------------------------------------
    |
    | A brief description of your API.
    |
    */
    'description' => 'This is the API documentation for our application.',
    
    /*
    |--------------------------------------------------------------------------
    | Groups
    |--------------------------------------------------------------------------
    |
    | Define your API endpoint groups here.
    |
    */
    'groups' => [
        'auth' => [
            'name' => 'Authentication',
            'description' => 'Endpoints for user authentication'
        ],
        'users' => [
            'name' => 'Users',
            'description' => 'Endpoints for managing users'
        ],
        // Add more groups as needed
    ],
];

Usage

1

Define Your Endpoints

Add the @api annotation to your controller methods:

/**
 * @api {get} /users Get all users
 * @apiGroup Users
 * @apiDescription Retrieve a list of all users
 *
 * @apiSuccess {Object[]} data List of users
 * @apiSuccess {Number} data.id User ID
 * @apiSuccess {String} data.name User name
 * @apiSuccess {String} data.email User email
 */
public function index()
{
    // Your controller logic
}
2

Generate Documentation

Run the command to generate documentation from your annotated code:

php artisan api-doc:generate
3

View Documentation

Visit the documentation page in your browser:

Your documentation will be available at http://your-app.test/api-docs (or whatever route prefix you configured).

Endpoint Definition

You can define endpoints using annotations in your controller methods. Here's a comprehensive example:

/**
 * @api {post} /users Create a new user
 * @apiGroup Users
 * @apiDescription Create a new user account
 *
 * @apiHeader {String} Authorization Bearer access token
 * @apiHeader {String} Content-Type application/json
 *
 * @apiParam {String} name User's full name
 * @apiParam {String} email User's email address
 * @apiParam {String} password User's password
 * @apiParam {String} [phone] User's phone number (optional)
 *
 * @apiSuccess {Object} data User object
 * @apiSuccess {Number} data.id User ID
 * @apiSuccess {String} data.name User name
 * @apiSuccess {String} data.email User email
 * @apiSuccess {String} data.created_at Registration date
 *
 * @apiError {Object} error Error object
 * @apiError {String} error.message Error message
 *
 * @apiExample {json} Request Example:
 *     {
 *       "name": "John Doe",
 *       "email": "john@example.com",
 *       "password": "secret"
 *     }
 *
 * @apiExample {json} Success Response:
 *     {
 *       "data": {
 *         "id": 1,
 *         "name": "John Doe",
 *         "email": "john@example.com",
 *         "created_at": "2023-05-15T10:00:00.000000Z"
 *       }
 *     }
 *
 * @apiExample {json} Error Response:
 *     {
 *       "error": {
 *         "message": "The email has already been taken."
 *       }
 *     }
 */
public function store(Request $request)
{
    // Your controller logic
}

Testing Interface

The package includes a built-in testing interface that allows you to try out your API endpoints directly from the documentation.

Features

  • Send requests with different HTTP methods
  • Add headers and query parameters
  • Test with different body types (JSON, form data, etc.)
  • Authentication support (Bearer token, Basic Auth, API keys)
  • View response headers and status codes

How to Use

  1. Navigate to an endpoint in your documentation
  2. Click on the "Request" tab in the testing interface
  3. Configure your request parameters, headers, and body
  4. Click the "Send" button to execute the request
  5. View the response in the "Response" tab

Note

The testing interface makes actual HTTP requests to your API, so ensure your API is running and accessible when testing.

Customization

You can customize the appearance and behavior of your API documentation in several ways:

1

Custom Views

Publish the views to customize the documentation layout:

php artisan vendor:publish --tag=api-doc-generator-views

This will copy the package views to resources/views/vendor/api-doc-generator where you can modify them.

2

Custom Styles

Add your own CSS to customize the appearance:

// In your custom CSS file
.api-doc-header {
    background: linear-gradient(135deg, #your-color-1 0%, #your-color-2 100%);
}

.api-doc-endpoint {
    border-color: #your-border-color;
}
3

Custom Groups

Define custom groups in the configuration file:

// In config/api-doc-generator.php
'groups' => [
    'custom' => [
        'name' => 'Custom Endpoints',
        'description' => 'My custom API endpoints',
        'color' => '#FF5733', // Optional custom color
    ],
],

Examples

Basic Endpoint Example

/**
 * @api {get} /posts Get all posts
 * @apiGroup Posts
 * @apiDescription Retrieve a paginated list of blog posts
 *
 * @apiParam {Number} [page=1] Page number
 * @apiParam {Number} [limit=10] Number of items per page
 *
 * @apiSuccess {Object[]} data List of posts
 * @apiSuccess {Number} data.id Post ID
 * @apiSuccess {String} data.title Post title
 * @apiSuccess {String} data.content Post content
 * @apiSuccess {String} data.author Post author
 * @apiSuccess {String} data.created_at Post creation date
 * @apiSuccess {Object} meta Pagination metadata
 * @apiSuccess {Number} meta.current_page Current page number
 * @apiSuccess {Number} meta.total Total number of items
 * @apiSuccess {Number} meta.per_page Number of items per page
 */
public function index(Request $request)
{
    $limit = $request->get('limit', 10);
    $posts = Post::paginate($limit);
    
    return response()->json([
        'data' => $posts->items(),
        'meta' => [
            'current_page' => $posts->currentPage(),
            'total' => $posts->total(),
            'per_page' => $posts->perPage(),
        ]
    ]);
}

Endpoint with Authentication

/**
 * @api {post} /posts Create a new post
 * @apiGroup Posts
 * @apiDescription Create a new blog post (requires authentication)
 *
 * @apiHeader {String} Authorization Bearer {token}
 *
 * @apiParam {String} title Post title
 * @apiParam {String} content Post content
 *
 * @apiSuccess {Object} data Newly created post
 * @apiSuccess {Number} data.id Post ID
 * @apiSuccess {String} data.title Post title
 * @apiSuccess {String} data.content Post content
 * @apiSuccess {String} data.author Post author
 * @apiSuccess {String} data.created_at Post creation date
 *
 * @apiError {Object} error Error object
 * @apiError {String} error.message Error message
 * @apiError {String} error.code Error code
 */
public function store(Request $request)
{
    $user = auth()->user();
    
    $validated = $request->validate([
        'title' => 'required|string|max:255',
        'content' => 'required|string',
    ]);
    
    $post = new Post();
    $post->title = $validated['title'];
    $post->content = $validated['content'];
    $post->author = $user->name;
    $post->save();
    
    return response()->json([
        'data' => $post
    ], 201);
}

Frequently Asked Questions

Q: How do I add authentication to my API documentation?

A: You can configure authentication in the package configuration file. The testing interface supports Bearer tokens, Basic Auth, and API keys.

Q: Can I use this package with Lumen?

A: Yes, the package is compatible with both Laravel and Lumen frameworks.

Q: How do I organize endpoints into groups?

A: You can use the @apiGroup annotation in your endpoint definitions and configure the groups in the configuration file.

Q: Can I customize the look and feel of the documentation?

A: Yes, you can publish the views and assets to customize the documentation to match your brand.

Q: Does this package support OpenAPI/Swagger?

A: Currently, the package uses its own annotation format, but OpenAPI/Swagger support is planned for a future release.
