Introduction

Laravel API Doc Generator automatically builds beautiful, interactive API documentation from your controller annotations. It includes a built-in testing interface so developers can try endpoints right from the docs.

Easy Setup

Automatic endpoint discovery and documentation generation in minutes.

Interactive Testing

Send requests, set headers, and inspect responses in the UI.

Customizable

Tweak templates, colors, and groups to fit your brand.

Installation

1

Install via Composer

Run the following command in your terminal:

composer require alagiesinghateh/laravel-api-doc-generator
2

Publish Assets (views, config)

Publish package assets to your application:

php artisan vendor:publish --provider="Alagiesinghateh\LaravelApiDocGenerator\ApiDocGeneratorServiceProvider" --tag="views"
php artisan vendor:publish --provider="Alagiesinghateh\LaravelApiDocGenerator\ApiDocGeneratorServiceProvider" --tag="config"
3

Secure the Docs (optional)

Enable auth or IP whitelisting in config/api-doc-generator.php.

Configuration

Tweak routes, groups, and metadata in config/api-doc-generator.php:

return [
    'route_prefix' => 'api-docs',
    'middleware'   => ['web'],
    'title'        => 'API Documentation',
    'description'  => 'This is the API documentation for our application.',

    'groups' => [
        'auth' => [
            'name' => 'Authentication',
            'description' => 'Endpoints for user authentication',
        ],
        'users' => [
            'name' => 'Users',
            'description' => 'Endpoints for managing users',
        ],
    ],
];

Usage

1

Annotate Controller Methods

Add @api annotations:

/**
 * @api {get} /users Get all users
 * @apiGroup Users
 * @apiDescription Retrieve a list of all users
 *
 * @apiSuccess {Object[]} data List of users
 * @apiSuccess {Number} data.id User ID
 * @apiSuccess {String} data.name User name
 * @apiSuccess {String} data.email User email
 */
public function index() {
    // Your controller logic
}
2

Generate Docs

Create docs from annotations:

php artisan api-doc:generate
3

Open in Browser

Docs available at http://your-app.test/api-docs (or your configured route_prefix).

Endpoint Definition

Define endpoints with headers, params, examples, and responses:

/**
 * @api {post} /users Create a new user
 * @apiGroup Users
 * @apiDescription Create a new user account
 *
 * @apiHeader {String} Authorization Bearer access token
 * @apiHeader {String} Content-Type application/json
 *
 * @apiParam {String} name User's full name
 * @apiParam {String} email User's email address
 * @apiParam {String} password User's password
 * @apiParam {String} [phone] User's phone number (optional)
 *
 * @apiSuccess {Object} data User object
 * @apiSuccess {Number} data.id User ID
 * @apiSuccess {String} data.name User name
 * @apiSuccess {String} data.email User email
 * @apiSuccess {String} data.created_at Registration date
 *
 * @apiError {Object} error Error object
 * @apiError {String} error.message Error message
 *
 * @apiExample {json} Request:
 * {
 *   "name": "John Doe",
 *   "email": "john@example.com",
 *   "password": "secret"
 * }
 *
 * @apiExample {json} Success:
 * {
 *   "data": {
 *     "id": 1,
 *     "name": "John Doe",
 *     "email": "john@example.com",
 *     "created_at": "2023-05-15T10:00:00Z"
 *   }
 * }
 *
 * @apiExample {json} Error:
 * {
 *   "error": {
 *     "message": "The email has already been taken."
 *   }
 * }
 */
public function store(Request $request) {
    // Your controller logic
}

Pro Tip

You can also define endpoints programmatically by extending your package's config and readers (e.g., merging FormRequest rules automatically).

Testing Interface

Try endpoints directly from your docs with real HTTP requests.

Features

  • Send requests with different HTTP methods
  • Add headers and query parameters
  • Test with JSON or form data
  • Authentication: Bearer, Basic, API keys
  • View response headers & status codes

How to Use

  1. Open any endpoint in the docs
  2. Switch to the Request tab
  3. Configure method, headers, params, and body
  4. Click Send to execute
  5. Inspect the Response tab

Note

The tester makes real requests to your API. Ensure your API is running and accessible.

Customization

Customize views, styles, and groups to match your brand.

1

Custom Views

Publish the views:

php artisan vendor:publish --provider="Alagiesinghateh\LaravelApiDocGenerator\ApiDocGeneratorServiceProvider" --tag="views"

Views will be copied to resources/views/vendor/api-doc-generator.

2

Custom Styles

Add your own CSS:

/* In your custom CSS file */
.api-doc-header {
  background: linear-gradient(135deg, #7c3aed 0%, #4f46e5 100%);
}
.api-doc-endpoint {
  border-color: #10b981;
}
3

Custom Groups

Define groups in the config:

'groups' => [
  'custom' => [
    'name' => 'Custom Endpoints',
    'description' => 'My custom API endpoints',
    'color' => '#FF5733',
  ],
],

Examples

Basic Endpoint

/**
 * @api {get} /posts Get all posts
 * @apiGroup Posts
 * @apiDescription Retrieve a paginated list of blog posts
 *
 * @apiParam {Number} [page=1] Page number
 * @apiParam {Number} [limit=10] Number of items per page
 *
 * @apiSuccess {Object[]} data List of posts
 * @apiSuccess {Number} data.id Post ID
 * @apiSuccess {String} data.title Post title
 * @apiSuccess {String} data.content Post content
 * @apiSuccess {String} data.author Post author
 * @apiSuccess {String} data.created_at Post creation date
 * @apiSuccess {Object} meta Pagination metadata
 * @apiSuccess {Number} meta.current_page Current page number
 * @apiSuccess {Number} meta.total Total number of items
 * @apiSuccess {Number} meta.per_page Number of items per page
 */
public function index(Request $request)
{
    $limit = $request->get('limit', 10);
    $posts = Post::paginate($limit);

    return response()->json([
        'data' => $posts->items(),
        'meta' => [
            'current_page' => $posts->currentPage(),
            'total' => $posts->total(),
            'per_page' => $posts->perPage(),
        ]
    ]);
}

Endpoint with Authentication

/**
 * @api {post} /posts Create a new post
 * @apiGroup Posts
 * @apiDescription Create a new blog post (requires authentication)
 *
 * @apiHeader {String} Authorization Bearer {token}
 *
 * @apiParam {String} title Post title
 * @apiParam {String} content Post content
 *
 * @apiSuccess {Object} data Newly created post
 * @apiSuccess {Number} data.id Post ID
 * @apiSuccess {String} data.title Post title
 * @apiSuccess {String} data.content Post content
 * @apiSuccess {String} data.author Post author
 * @apiSuccess {String} data.created_at Post creation date
 *
 * @apiError {Object} error Error object
 * @apiError {String} error.message Error message
 * @apiError {String} error.code Error code
 */
public function store(Request $request)
{
    $user = auth()->user();

    $validated = $request->validate([
        'title' => 'required|string|max:255',
        'content' => 'required|string',
    ]);

    $post = new Post();
    $post->title = $validated['title'];
    $post->content = $validated['content'];
    $post->author = $user->name;
    $post->save();

    return response()->json(['data' => $post], 201);
}
